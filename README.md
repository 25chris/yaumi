# Yaumi - Daily Prayer Activity and Attendance App

Welcome to the repository for **Yaumi**, an Android application designed to manage daily prayer activities and attendance for companies, schools, and similar communities. Yaumi offers a comfortable and efficient way to track member attendance and ensure active participation in daily prayers.

## About the Project

Yaumi is developed to provide a seamless solution for managing attendance and prayer activities in various community settings. It aims to simplify the process of tracking participation and maintaining records.

### Features

- **Daily Prayer Schedule**: Provides a daily schedule for prayer times.
- **Attendance Tracking**: Easy and accurate tracking of member attendance.
- **Member Management**: Manage member information and attendance records.
- **Notifications**: Send reminders and notifications for prayer times and attendance.
- **Reports**: Generate attendance reports for analysis and record-keeping.

## Technology Stack

Yaumi is built using modern technologies to ensure a smooth and efficient user experience on Android devices.

### Key Technologies Used:

- **Flutter**: A UI toolkit for crafting natively compiled applications for mobile.
- **Firebase**: Backend services for authentication, real-time database, and notifications.
- **Provider**: State management for handling app state efficiently.
- **Dart**: Programming language used with Flutter.

## Getting Started

To get a local copy up and running, follow these simple steps.

### Prerequisites

- Flutter SDK: [Installation Guide](https://flutter.dev/docs/get-started/install)
- Dart SDK: Included with Flutter SDK
- Android Studio: [Download and Install Android Studio](https://developer.android.com/studio)
- Git: [Download and Install Git](https://git-scm.com/downloads)

### Installation

1. Clone the repo
   ```sh
   git clone https://github.com/25chris/yaumi.git
   ```
2. Navigate to the project directory
   ```sh
   cd yaumi
   ```
3. Install dependencies
   ```sh
   flutter pub get
   ```
4. Run the app on an emulator or physical device
   ```sh
   flutter run
   ```
